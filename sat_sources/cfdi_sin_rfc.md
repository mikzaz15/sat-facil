# CFDI sin RFC del receptor: criterios y manejo operativo

## 1) Objetivo

Orientar sobre escenarios donde el receptor no proporciona RFC y como manejar la emision con criterios consistentes.

## 2) Definiciones

- RFC del receptor: dato fiscal clave para timbrado en CFDI 4.0.
- Publico en general: esquema operativo usado en ciertos casos de venta masiva.
- Datos minimos: conjunto de campos necesarios para emitir sin rechazo.

## 3) Reglas SAT operativas

1. Priorizar siempre obtener RFC del receptor cuando sea posible.
2. Si no se cuenta con RFC, aplicar el procedimiento permitido para el caso especifico.
3. Capturar datos obligatorios del receptor conforme la regla aplicable.
4. Mantener evidencia de que el receptor no proporciono RFC cuando corresponda.

## 4) Escenarios comunes

### Escenario A: venta mostrador sin RFC

- Aplicar flujo permitido para publico en general (segun reglas vigentes).
- Cuidar consistencia de datos generales y totales.

### Escenario B: cliente solicita factura despues

- Definir politica interna de plazo y recaptura de datos.
- Evitar doble facturacion de la misma venta.

### Escenario C: error por RFC mal capturado

- Evaluar cancelacion/sustitucion segun proceso interno y regla vigente.

## 5) Ejemplos practicos

### Ejemplo 1

- Cliente no proporciona RFC al momento.
- Se aplica esquema permitido para operacion de mostrador.
- Se conserva ticket y evidencia para trazabilidad.

### Ejemplo 2

- Cliente envia RFC despues del corte.
- Se aplica politica de refacturacion controlada evitando duplicidad.

## 6) Errores comunes

- Inventar RFC para timbrar rapido.
- No conservar evidencia de la operacion original.
- No conciliar ticket/venta con CFDI final.

## 7) Controles recomendados

1. Politica formal para facturas sin RFC en punto de venta.
2. Validaciones de duplicidad por ticket/folio interno.
3. Bitacora de correcciones y sustituciones.
4. Capacitacion del personal de caja/facturacion.

## 8) Checklist

1. Intento de obtener RFC registrado.
2. Flujo aplicable identificado (receptor especifico o publico en general).
3. Datos obligatorios completos.
4. Evidencia de venta conservada.
5. Conciliacion contable cerrada.

## 9) Referencias SAT sugeridas

- Guia de llenado CFDI 4.0 para datos del receptor.
- Reglas SAT vigentes para operaciones con publico en general.
- Anexo 20 vigente.
- Criterios SAT aplicables a cancelacion/sustitucion por datos incorrectos del receptor.
